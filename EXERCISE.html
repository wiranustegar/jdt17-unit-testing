Skenario

Kamu baru bergabung di tim Employee Management di Indivara. Tim sebelumnya sudah menulis production code yang lengkap, tapi unit test-nya baru tercover sebagian. Tugasmu adalah melengkapi semua unit test yang hilang agar coverage mencapai 100%.

⚠ Kebijakan PMO Indivara
Di Indivara, unit test harus mencapai 100% LINE dan BRANCH coverage agar lolos quality gate dan bisa di-deploy ke production. Code yang coverage-nya di bawah 100% akan ditolak oleh PMO dan tidak bisa masuk ke environment production. JaCoCo sudah dikonfigurasi di pom.xml project ini (cek LINE + BRANCH = 1.00) — jika salah satu belum 100%, mvn test akan GAGAL (BUILD FAILURE).

Pola yang Harus Diikuti

1. Pola AAA (Arrange → Act → Assert)

Arrange
Siapkan data & mock
→
Act
Panggil method
→
Assert
Cek hasilnya

2. Golden Rule: Positive + Negative

Setiap positive test harus punya minimal 1 negative test sebagai pasangannya.
Contoh: getById_WhenExists (positive) ↔ getById_WhenNotExists (negative)

3. Gunakan verify() untuk Side Effects

// Pastikan repository.save() dipanggil tepat 1 kali
verify(employeeRepository, times(1)).save(any(Employee.class));

// Pastikan repository.save() TIDAK PERNAH dipanggil (skenario error)
verify(employeeRepository, never()).save(any(Employee.class));

4. Naming Convention

// Format: methodName_Condition_ExpectedBehavior
void getEmployeeById_WhenNotExists_ShouldThrowException()
void createEmployee_WithValidData_ShouldReturnCreated()
void deleteEmployee_WhenExists_ShouldDeleteSuccessfully()

Tips & Petunjuk

Service Test — Ingat:

  • @Mock = tiruan (bukan koneksi asli ke database)
  • @InjectMocks = instance asli dari service, tapi dependency-nya mock
  • Konfigurasi mock: when(repo.method()).thenReturn(value)
  • Untuk method void: doNothing().when(repo).delete(obj)
  • Untuk test exception: assertThrows(ExceptionClass.class, () -> ...)
  • Selalu akhiri dengan verify() untuk cek interaksi

Controller Test — Ingat:

  • Gunakan mockMvc.perform(get/post/put/delete(...))
  • Untuk POST/PUT: tambahkan .contentType(MediaType.APPLICATION_JSON) dan .content(objectMapper.writeValueAsString(obj))
  • Cek status: .andExpect(status().isOk()), .isCreated(), .isNotFound(), .isBadRequest(), .isNoContent(), .isConflict()
  • Cek JSON: .andExpect(jsonPath("$.name").value("Budi"))
  • Validation error → Spring otomatis return 400 sebelum memanggil service
Perhatikan perbedaan:
  • 400 Bad Request = validasi gagal (nama kosong, salary negatif) → service TIDAK dipanggil
  • 404 Not Found = data tidak ditemukan → service melempar EmployeeNotFoundException
  • 409 Conflict = email duplikat → service melempar DuplicateEmailException

Cara Menjalankan

1. Import Project

Buka folder practice/ di IntelliJ IDEA atau VS Code. Maven akan otomatis download dependencies.

2. Tulis Test

Buka file test, cari komentar TODO #1, TODO #2, dst. Tulis test sesuai petunjuk dan hint yang diberikan.

3. Jalankan Test

# Jalankan semua test
mvn test

# Jalankan hanya Service test
mvn test -Dtest=EmployeeServiceTest

# Jalankan hanya Controller test
mvn test -Dtest=EmployeeControllerTest

4. Cek Hasil Test

5. Cek Coverage Report (JaCoCo)

Setelah mvn test selesai, JaCoCo otomatis generate report HTML. Buka di browser:

open target/site/jacoco/index.html       # macOS
start target/site/jacoco/index.html      # Windows

Cara Membaca Report

Di halaman utama ada 2 kolom penting: Lines (LINE coverage) dan Missed Branches (BRANCH coverage). Klik nama class untuk lihat warna per baris:

Tanda di ReportArtinyaAksi
Baris HijauBaris ini dieksekusi oleh test (LINE ✓)Sudah aman ✔
Baris MerahBaris ini BELUM pernah dieksekusi (LINE ✗)Tambahkan test!
Berlian ◆ KuningBRANCH sebagian — baru sebagian cabang if/&& diuji (mis. "2 of 4 branches missed")Test kondisi TRUE & FALSE-nya
Berlian ◆ HijauBRANCH penuh — semua cabang sudah diujiSudah aman ✔
Jika coverage belum 100%:
  1. Buka report → klik nama class yang coverage-nya belum 100%
  2. Cari baris merah (LINE belum) dan berlian ◆ kuning (BRANCH belum)
  3. Tulis test baru: untuk merah → eksekusi barisnya; untuk kuning → uji kondisi TRUE dan FALSE-nya
  4. Jalankan mvn test lagi
  5. Ulangi sampai semua baris hijau & semua berlian hijau dan build SUCCESS
